Проект реализует консольную C++-программу для проверки алгоритма бота, который исследует подземелье, собирает ресурсы и возвращается в стартовую комнату. Описание подземелья читается из входного файла, последовательность действий записывается в `result.txt`.

## Правила симуляции

- Комнаты нумеруются от `0` до `N`; комната `0` — стартовая.
- В комнатах, кроме стартовой, могут находиться ресурсы: `iron`, `gold`, `gems`, `exp`.
- Базовая ценность ресурсов:

| Ресурс | Ценность |
| ------ | -------- |
| `iron` | 7        |
| `gold` | 11       |
| `gems` | 23       |
| `exp`  | 1        |

- На входе задаётся целевой ресурс, ценность которого в текущем подземелье удваивается.
- Еда расходуется на перемещение между комнатами и на повторные сборы ресурсов в одной комнате.
- Первый сбор ресурса в каждой комнате не расходует еду.
- Если еда закончилась не в стартовой комнате, персонаж считается погибшим; реализованный алгоритм планирует возврат с учётом оставшегося запаса.

## Алгоритм бота

Симуляция состоит из двух фаз.

### 1. Фаза исследования

Бот тратит половину стартового запаса еды на исследование:

1. Переходит в смежную непосещённую комнату с наименьшим номером.
2. Если все смежные комнаты уже посещены, выбирает ближайшую непосещённую комнату с наименьшим номером.
3. При первом посещении комнаты собирает ресурс с максимальной ценностью.

### 2. Фаза возвращения

Бот возвращается в комнату `0`:

1. Идёт по кратчайшему пути среди посещённых комнат.
2. При равных вариантах на развилке выбирает комнату с наименьшим номером.
3. Если еды больше, чем нужно для возврата, тратит излишки на сбор оставшихся ресурсов на обратном пути, начиная с наиболее ценных.

## Структура проекта

```text
.
├── Bot.h / Bot.cpp          # алгоритм бота и симуляция действий
├── Cave.h / Cave.cpp        # модель подземелья, парсинг и валидация входных данных
├── ParseError.h             # исключение с исходной некорректной строкой
├── main.cpp                 # CLI-точка входа: чтение файла и запись result.txt
├── CMakeLists.txt           # CMake-сборка библиотеки и executable
└── README.md
```

CMake собирает:

- библиотеку `dungeon_simulator` из `Bot.cpp` и `Cave.cpp`;
- исполняемый файл `yadro_telecom` из `main.cpp`.

Такое разделение упрощает интеграцию альтернативных алгоритмов бота и подключение симулятора к тестам.

## Требования

- C++17-совместимый компилятор;
- CMake версии `3.16` или новее.

## Сборка

Рекомендуемый способ — out-of-source сборка через CMake:

```bash
cmake -S . -B build
cmake --build build
```

После сборки исполняемый файл будет находиться в директории `build`:

```bash
./build/yadro_telecom <input-file>
```

Также проект можно собрать напрямую компилятором, например:

```bash
g++ -std=c++17 main.cpp Cave.cpp Bot.cpp -o yadro_telecom
```

## Запуск

```bash
./build/yadro_telecom path/to/input.txt
```

Программа принимает один аргумент — путь к файлу с описанием подземелья. Результат всегда записывается в файл `result.txt` в текущей рабочей директории.

Если аргумент не передан или входной файл не удалось открыть, сообщение об ошибке выводится в `stderr`.

## Формат входных данных

Первая строка:

```text
N
```

где `N` — количество комнат без учёта стартовой комнаты `0`, допустимый диапазон: `[1, 255]`.

Далее идут `N + 1` строк описания комнат:

```text
<номер комнаты> <список смежных комнат через запятую> <iron> <gold> <gems> <exp>
```

Количество каждого ресурса должно быть в диапазоне `[0, 255]`.

Последняя строка:

```text
<количество еды> <iron|gold|gems|exp>
```

где количество еды находится в диапазоне `[2, 255]`, а второй параметр задаёт ресурс с удвоенной ценностью.

## Формат выходных данных

При перемещении:

```text
go <номер комнаты>
```

При сборе ресурса:

```text
collect <iron|gold|gems|exp>
```

После каждого действия, кроме финального возврата в стартовую комнату, выводится состояние текущей комнаты:

```text
state <номер комнаты> <iron> <gold> <gems> <exp>
```

Собранный ресурс отображается символом `_`.

После возврата в стартовую комнату выводится итог:

```text
result <iron> <gold> <gems> <exp> <общая ценность>
```

## Пример

Входной файл:

```text
2
0 1 0 0 0 0
1 0,2 1 2 0 3
2 1 0 1 1 0
4 gems
```

Запуск:

```bash
./build/yadro_telecom input.txt
```

Содержимое `result.txt`:

```text
go 1
state 1 1 2 0 3
collect gold
state 1 1 _ 0 3
go 2
state 2 0 1 1 0
collect gems
state 2 0 1 _ 0
go 1
state 1 1 _ 0 3
go 0
result 0 2 1 0 68
```

## Валидация входных данных

При некорректных данных программа записывает в `result.txt` исходную строку, на которой обнаружена ошибка, и завершает работу с ненулевым кодом возврата.
